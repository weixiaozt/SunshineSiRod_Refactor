# ABCD 边长独立检测块：长棒模板标定长度算法

本目录只开发和验证 `A/B/C/D` 四个边长。端面、对角线、倒角、投影、棒长和 Web
接入均不属于本模块，后续按独立检测块分别开发。

## 最高规则：禁止根据名称或路径判断调头

程序不得读取、搜索或解释 HOBJ 文件名、父目录名、文档名中的以下词语：

```text
正、反、正向、反向、调头、掉头、转向、diaotou、turnover、reverse
```

这些词只说明实验中存在过“把同一根棒换姿态后再次拍摄”的情况，不是测量输入，也不
代表程序知道当前被测物是否调头。现实生产中没有外部方向传感器时，程序不知道棒子是否
被调头，因此：

- 所有 HOBJ 必须执行同一套坐标转换、面号定义、边长公式和汇总规则；
- 不得因路径或文件名切换标定模板；
- 不得执行 `A/B/C/D` 交换、头尾交换或 `180°-角度` 等路径驱动映射；
- 不得根据名称选择更接近标准值的一套结果；
- 模型文件不得包含运行时方向分类器或方向关键词规则。

调头拍摄只用于验证：同一物体改变摆放姿态后，算法输出是否仍准确、重复和自洽。六张图在拟合
唯一相机模型时仍是六次等权观测；另外使用人工明确编写的
`calibration_capture_manifest.csv` 做标定阶段自洽性审计，并从三对已知物理姿态的
`B-D` 偶对称分量推导唯一设备链相对偏差 `K`。该清单不是从文件名推理出来的，绝不
进入单图测量、运行时方向判断或模型切换。

已知物理调头验证关系为：

```text
A ↔ A
C ↔ C
B ↔ D
head ↔ tail
```

相机在设备上的位置始终不变；变的是调头后同一物理 B/D 面由另一侧相机链观测。

## 完美棒定义

真值文件为 `CTB-长度数据.csv`：

```text
A = C = 210.01 mm
B = D = 105.11 mm
```

这表示标定阶段把长 CTB 的理想横截面定义为完美矩形。它的用途是建立四相机局部坐标到
同一截面坐标的空间关系，不是给生产结果增加四个最终边长偏移。生产 HOBJ 的每个 A/B/C/D
都必须由当前图像的四个角点重新计算，不裁剪到规格范围，不强制 `A=C` 或 `B=D`。

## 六图标定流程

1. 枚举本目录下六张 HOBJ，只按稳定排序读取，不分析名称含义。
2. 求六图四相机共同有效的绝对设备行范围，在两端保留安全边界后默认取 850 个标定站点，约每 1 mm 一片。
3. 每台相机在六图共同可见的主面窗口内提取角点局部坐标与两侧主面斜率。
4. 六图共同拟合每台相机的正交局部坐标矩阵。
5. 每个绝对行、每台相机分别得到“完美矩形角点 − 当前局部角点”的空间原点观测。
6. 对六次观测做鲁棒中位数，再做短窗口平滑，压制局部毛刺和棒身微小高频波动；不根据
   拍摄方向分组或切换模板。
7. 六次留一：每次连共同ROI窗口、坐标矩阵和逐行原点都只用五图建立，再测量完全未参与
   建模的第六图，记录四边逐片 RMS 与最终误差。
8. 只有模型结构完整、六图哈希可追溯且留一误差通过门限，模型才标记为 `valid=true`。

基础六图模型的 `valid=true` 只表示同一根标准棒内部验证通过。正式的“长棒模板标定长度
算法”另外固定启用相机Y同步、单一B/D设备链K和头尾密集平均。K先在CTB、CMM、GTT
三根长棒内分别按明确的实验分组求B/D交叉自洽值，再让三根棒等权平均；CMM已知碎角图
`14_16.hobj`明确排除。正式模型文件为
`models/long_rod_template_calibrated_length_210_105.json`。
旧的`abcd_length_final_y_sync_k_head_tail_210_105.json`仅保留作CTB单棒K审计，已经
不是默认模型，不得作为“长棒模板标定长度算法”继续使用。

## 单张 HOBJ 测量流程

每个有效标定站点只做 ABCD：

```text
相机 Top  -> 左上角
相机 Right -> 右上角
相机 Left  -> 右下角
相机 Down  -> 左下角

A = distance(Top, Right)
B = distance(Right, Left)
C = distance(Left, Down)
D = distance(Down, Top)
```

每张图默认计算 850 个有效站点。先按固定四相机Y偏移插值到共同物理Y，再计算每片
ABCD。最终汇总为：

```text
A/C 最终值 = 全部有效站点的算术平均
B/D raw = (头部 100 片平均 + 尾部 100 片平均) / 2

B final = B raw + K/2
D final = D raw - K/2
K = -0.007022301417 mm
```

K是CTB、CMM和GTT三根长棒等权得到的设备相机链相对偏差，不是四边独立终值补偿。
A/C不变，B与D等量反向修正，
每片均保持 `B+D` 不变。输出同时保留每片raw/corrected、raw头尾/全局汇总和最终汇总；
`reported_edges_mm` 固定使用 A/C 全局值与修正后的B/D头尾密集平均。

## 目录结构

```text
build_calibration.py        构建六图标定模型并执行留一验证
build_final_abcd_model.py   从固定Y基础模型及CTB/CMM/GTT明确分组推导K并构建正式模型
measure_abcd.py             使用模型测量单张 HOBJ
calibration_capture_manifest.csv  人工明确的三对标定姿态，仅验证用
validate_six.py             六图重复性和 A/C、B/D 调头自洽性验证
src/abcd_length/            独立算法实现
tests/                      单元测试
models/                     生成的标定模型
results/                    留一诊断和测量结果
```

当前阶段不修改 Web 配置，也不打包 ZIP。

## 当前命令

```powershell
python build_calibration.py
python build_final_abcd_model.py
python measure_abcd.py --input "一张图.hobj" --output "results\\one.json"
python validate_six.py
```

`validate_six.py` 同时输出六图总体重复性与人工清单指定的三对姿态自洽性。
对 A/C 比较同名全局值；对 B/D 分别比较全局 `B↔D` 和头尾 `head↔tail, B↔D`。
这些是标定后验证，不会调用另一套测量逻辑。
