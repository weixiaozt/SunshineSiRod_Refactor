# 长棒倒角几何算法

本目录是独立的倒角几何测量模块，暂不接入 Web。输入一个四相机 `.hobj`，输出
`AD、AB、CB、CD` 四个角点各 4 个值，共 16 个最终值。

## 几何定义

截面和固定相机关系：

```text
P3 / AD / Camera3-Top ----- A ----- P1 / AB / Camera1-Right
          |                                  |
          D                                  B
          |                                  |
P2 / CD / Camera2-Down ---- C ----- P4 / CB / Camera4-Left
```

每个角点的一条切片轮廓拟合三条线：

- `L1、L2`：倒角两侧的实体主面拟合线；
- `Lc`：自适应识别出的倒角段拟合线；
- `P = intersection(L1, L2)`：主面延长线形成的理论尖角；
- `T1 = intersection(L1, Lc)`、`T2 = intersection(L2, Lc)`：两个倒角端点。

由此直接计算：

```text
倒角弦长 = |T1 - T2|
主面1投影 = |P - T1|
主面2投影 = |P - T2|
尖角夹角 = angle(T1 - P, T2 - P)，范围 0°～180°
```

“倒角长度”是两个端点之间的直线弦长，不是曲线积分弧长。两侧投影分别实算，
不再使用旧代码中的 `弦长 / sqrt(2)`。只有实际为对称 45° 倒角时，两侧投影才应
接近相等。

## 单切片算法

1. 在目标行前后各 2 行取逐列中位数，获得一条降噪轮廓。
2. 沿用交付代码已验证的镜像和 `500:3200` 裁剪约定，并按相机公制比例转换为 mm。
3. 平滑轮廓只用于定位角部峰值；所有直线仍使用原始公制轮廓点拟合。
4. 在峰值两侧宽窗口内用 MAD 迭代拟合两个主面。
5. 计算每个点到两条主面的垂距；同时离开两条主面的中央连续段就是倒角候选段。
6. 对倒角候选段鲁棒拟合 `Lc`，再按上面的交点公式计算 4 个值。
7. 任何切片拟合失败只记录失败原因，不修改其他切片的结果。

默认沿四相机共同有效棒身范围均匀取 850 片，去掉两端各 250 行。每个角点独立
保留有效切片，最终值使用 10% 截尾平均；同时保留有效数量、标准差、极差和逐切片
拟合残差，便于判断重复性和局部缺陷。

## 标定边界

需要的是低层、可物理追溯的相机公制标定：

- 相机横向像素到毫米的比例；
- 轮廓高度值到毫米的比例；
- HOBJ 四通道与 `AD/AB/CB/CD` 的固定相机及面号映射。

本算法不需要四角 16 个最终值偏移，也不使用长度算法的 B/D K。单个倒角全部在
同一相机局部坐标内计算，因此平移原点和刚体旋转不会改变弦长、投影或夹角；但 X/Z
非等比例误差会改变长度和角度，必须由物理标定保证。

## 调头与文件名铁律

目录名、文件名或文档中出现 `正、反、调头、转向、diaotou、反向` 等字样，只说明
测试素材中可能存在调头验证，不是程序可用的方向信号。生产测量对每个 HOBJ 始终运行
完全相同的四相机几何逻辑，不执行头尾交换、B/D 交换或角度补角。调头图只用于验证
算法的重复性和物理自洽性。

## 使用

仅在窗口显示最终结果：

```powershell
python .\measure_chamfer_geometry.py "D:\path\capture.hobj"
```

需要审计文件时显式指定：

```powershell
python .\measure_chamfer_geometry.py "D:\path\capture.hobj" `
  --json ".\results\capture_result.json" `
  --slice-csv ".\results\capture_slices.csv"
```

程序默认不写 CSV/JSON。当前阶段也不修改现有 Web 和离线 ZIP。

## v1：实体倒角端点对角线

新增的对角线程序不再使用倒角中点，也不沿用旧交付算法的有限区域最大支撑距离。
两个长度固定定义为：

```text
对角线1 = distance(AB倒角A侧端点/角点2, CD倒角C侧端点/角点6)
对角线2 = distance(AD倒角A侧端点/角点1, BC倒角C侧端点/角点5)
```

端点来自本目录倒角几何v3。跨相机距离不能直接用各自局部坐标相减，因此程序复用
“长棒模板标定长度算法”中已经固化的四相机全局X/Z矩阵、850个扫描位置原点曲线和
固定Y偏移，在共同物理Y上插值后逐片计算欧氏距离。长度算法中的B/D关系K不参与
对角线，程序也没有两条对角线终值偏移或文件路径方向判断。

窗口运行：

```powershell
python .\measure_diagonal_geometry.py "D:\path\capture.hobj"
```

调试时可用 `--stations 100` 减少切片；正式候选默认使用模型全部850片。输出同时保留
10%截尾均值、标准差、极差、头/中/尾均值和有效切片数。当前为第一版可运行候选，
在独立对角线真值和跨姿态重复性验证完成前，不宣称绝对精度正式放行。

## v3：标定域钳制的局部尺度指纹

固定二维矩阵能够修正每台相机的比例、非正交和剪切，但CMM回归显示，约
`1.10 mm` 与 `1.20 mm` 的两种倒角仍存在向中间压缩的尺度响应；在约
`1.95 mm` 的105-4同棒调头图上，这一问题表现为不同相机之间的弦长差增大。

v3保留v2的四个固定2×2矩阵，在每台相机的理论尖角射线上增加一个低层局部尺度曲线：

```text
delta = clamp(|ray|² - reference, calibrated_delta_min, calibrated_delta_max)
ray_corrected = ray_linear × (1 + coefficient × delta)
```

`reference、coefficient、calibrated_delta_min/max`均固定绑定相机。上下界来自CMM训练图
中该相机射线平方长度的P05/P95范围；生产值超出CMM覆盖范围时只使用边界修正，禁止
继续外推。因此它不是四角终值偏移，也不会根据HOBJ路径、文件名或是否调头切换参数。

放行依据记录在 `calibration/local_scale_fingerprint_acceptance_v3.json`。当前只放行弦长
和双投影的该项尺度修正；角度仍由同一组P/T端点和固定2×2矩阵直接计算，没有增加角度
终值补偿。仅有一组105-4大倒角同棒调头证据时，不足以安全识别新的角度非线性参数。
